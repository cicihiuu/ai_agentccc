# 7.2 文档验收声明

本文件用于声明项目书 7.2 文档验收要求在当前项目 Markdown 文档中的对应关系。

原项目书不修改。本文档作为项目内 Markdown 验收声明，说明各项文档要求由哪些交付文件满足。

## 文档验收矩阵

| 项目书要求 | 对应交付文档 | 覆盖章节 |
| --- | --- | --- |
| 完整的项目设计文档：包含架构设计、模块接口说明、数据库设计 | `docs/PROJECT_DESIGN.md` | `1. 架构设计`、`3. 模块接口说明`、`4. 数据库设计` |
| 用户使用手册：包含环境搭建、工具使用、参数说明 | `docs/USER_MANUAL.md` | `1. 环境搭建`、`3. 工具使用`、`4. 参数说明` |
| 开发文档：包含各模块的开发说明、代码注释规范 | `docs/DEVELOPER_GUIDE.md` | `2. 模块开发说明`、`8. 代码注释规范` |

## 验收方式

人工验收：

1. 打开 `docs/PROJECT_DESIGN.md`，检查是否包含架构设计、模块接口说明、数据库设计。
2. 打开 `docs/USER_MANUAL.md`，检查是否包含环境搭建、工具使用、参数说明。
3. 打开 `docs/DEVELOPER_GUIDE.md`，检查是否包含模块开发说明、代码注释规范。

自动验收：

```powershell
.\.venv\Scripts\python.exe -m unittest tests.test_docs_acceptance
```

## 说明

- 文档以 Markdown 作为最终验收交付物。
- 数据库设计按当前项目实际实现解释为文件态状态存储设计，并补充未来数据库表映射。
- 本声明不改变扫描、报告、前端页面或 API 行为。
